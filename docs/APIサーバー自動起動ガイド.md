# APIサーバー自動起動ガイド

このドキュメントは、FastAPIサーバーとTailscale FunnelをPC起動時に自動で立ち上げるための手順を説明します。

## 1. ファイル構成

自動起動は、以下の3つのバッチファイルによって実現されます。これらのファイルはプロジェクトのルートディレクトリに配置されています。

-   `start_api.bat`
    -   **親ランチャー**。このファイルを実行すると、他の2つのスクリプトがそれぞれ別ウィンドウで起動します。
-   `run_fastapi.bat`
    -   FastAPIサーバー（Uvicorn）を起動するための子スクリプト。
-   `run_funnel.bat`
    -   Tailscale Funnelを起動するための子スクリプト。

この分割構成により、Windows環境で安定した自動起動を実現しています。

## 2. 手動での起動方法

1.  プロジェクトフォルダにある `start_api.bat` をダブルクリックします。
2.  以下の2つのウィンドウが自動で開き、表示されたままになります。
    -   **FastAPI Server**: `Uvicorn running on http://0.0.0.0:10000` と表示されます。
    -   **Tailscale Funnel**: `https://<あなたのドメイン>.ts.net -> http://localhost:10000` のように、外部からアクセス可能な公開URLが表示されます。

この2つのウィンドウが開いている間、APIは外部から利用可能です。終了するには、各ウィンドウを個別に閉じてください。

## 3. PC起動時の完全自動化設定

毎回手動で実行する手間を省き、Windowsへのログイン時にAPIサーバーを自動で起動する設定です。

1.  `start_api.bat` ファイルを右クリックし、「**ショートカットの作成**」を選択します。
2.  キーボードで `Windowsキー + R` を押し、「ファイル名を指定して実行」ウィンドウを開きます。
3.  入力欄に `shell:startup` と入力して「OK」を押します。
4.  開いた「スタートアップ」フォルダに、先ほど作成した `start_api.bat` のショートカットを移動（ドラッグ＆ドロップ）します。

これで、次回からPCにログインするたびに、APIサーバーとFunnelが自動的にバックグラウンドで起動します。

### (オプション) 起動時のウィンドウを非表示にする

起動時にコマンドプロンプトのウィンドウが表示されるのが気になる場合は、ショートカットの設定で最小化実行が可能です。

1.  「スタートアップ」フォルダに置いたショートカットを右クリックし、「プロパティ」を選択します。
2.  「実行時の大きさ」のドロップダウンメニューから「**最小化**」を選択し、「OK」を押します。

これにより、起動時にウィンドウが一瞬タスクバーに表示されるだけで、デスクトップを占有しなくなります。

## 4. トラブルシューティング

-   **ウィンドウが一瞬で消えてしまう場合**:
    -   `run_fastapi.bat` または `run_funnel.bat` のどちらかの実行に失敗している可能性があります。
    -   各子バッチファイルを個別にダブルクリックして、表示されるエラーメッセージを確認してください。
    -   主な原因として、Python仮想環境（`.venv`）の破損や、Tailscaleのインストール場所が異なる、などが考えられます。
### ステップ1：タスクスケジューラを開く

1.  `Windowsキー + R` を押し、「ファイル名を指定して実行」ウィンドウを開きます。
2.  `taskschd.msc` と入力し、`Enter`キーを押します。

### ステップ2：タスクの作成

1.  右側の「操作」ペインから `タスクの作成...` をクリックします。

### ステップ3：「全般」タブの設定

1.  **名前**: `YouTube Summary API Auto Start` など、分かりやすい名前を入力します。
2.  **説明**: `PC起動時にFastAPIとTailscale Funnelを起動する` など、タスクの内容が分かる説明を入力します。
3.  **セキュリティオプション**:
    -   `◎ ユーザーがログオンしているかどうかにかかわらず実行する` を選択します。（これにより、PC起動直後、ログイン前からタスクが実行されます）
    -   `☑ 最上位の特権で実行する` にチェックを入れます。

### ステップ4：「トリガー」タブの設定

1.  `新規...` ボタンをクリックします。
2.  「タスクの開始」のドロップダウンメニューから `スタートアップ時` を選択します。
3.  `OK` をクリックします。

### ステップ5：「操作」タブの設定

1.  `新規...` ボタンをクリックします。
2.  「操作」が `プログラムの開始` になっていることを確認します。
3.  「プログラム/スクリプト」の欄に、`start_api.bat` のフルパスを入力します。
    -   例: `C:\Users\hirok\Documents\Windsurf\811【開発】\youtube_summary_api\start_api.bat`
    -   `参照...` ボタンからファイルを選択することも可能です。
4.  `OK` をクリックします。

### ステップ6：「条件」タブの設定

1.  「電源」セクションの `☑ タスクを実行するためにスリープを解除する` のチェックを外します。
2.  「ネットワーク」セクションの `☑ 次のネットワーク接続が使用可能な場合のみタスクを開始する` のチェックを外します。（これにより、オフライン状態でもバッチファイルの実行が試みられます）

### ステップ7：設定完了と保存

1.  すべての設定が終わったら `OK` をクリックしてタスクを保存します。
2.  この時、パスワードの入力を求められる場合があります。お使いの[[Windows]]アカウントのログインパスワードを入力してください。

## 確認方法

設定が完了したら、PCを再起動してください。
ログイン後、しばらくしてからWebブラウザで `http://localhost:10000/docs` にアクセスし、APIのドキュメントページが表示されれば、FastAPIサーバーは正常に起動しています。

また、[[Tailscale]]の管理画面やコマンドラインで、Funnelが有効になっていることを確認してください。

## トラブルシューティング

- **自動起動しない場合**: 
    - タスクスケジューラの対象タスクの「履歴」タブを確認し、エラーが出ていないか確認します。
    - `start_api.bat` を手動でダブルクリックして、正常にサーバーが起動するか確認します。
    - バッチファイル内のパスが正しいか再確認します。
- **コマンドプロンプトのウィンドウが一瞬で消える場合**:
    - `start_api.bat` の最後に `pause` と一行追加して、手動実行した際にエラーメッセージが表示されていないか確認してください。確認後、`pause` は削除してください。

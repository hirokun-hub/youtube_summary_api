# PC起動時のAPIサーバー自動起動手順

このドキュメントは、[[Windows]] PCの起動時に、開発した[[FastAPI]]サーバーと[[Tailscale Funnel]]を自動的に起動させるための設定手順を説明します。
これにより、PCを再起動するたびに手動でサーバーを立ち上げる手間を省くことができます。

## 概要

[[Windows]]に標準で搭載されている「[[タスクスケジューラ]]」という機能を利用します。
[[タスクスケジューラ]]に「PC起動時」というトリガーで、APIサーバーとFunnelを起動するバッチファイル(`start_api.bat`)を実行するタスクを登録します。

## 前提条件

- プロジェクトルートに `start_api.bat` ファイルが配置されていること。
- `start_api.bat` 内の各種パス（仮想環境、プロジェクトディレクトリなど）が、ご自身のPC環境に合わせて正しく設定されていること。

### `start_api.bat` の内容確認

念のため、`start_api.bat` の内容が以下のようになっているか確認してください。
パスが異なる場合は、ご自身の環境に合わせて修正してください。

```bat
@echo off
REM --- APIサーバー兼Tailscale Funnel起動バッチ ---

REM --- 設定 --- 
REM プロジェクトのルートディレクトリを指定
SET PROJECT_DIR="C:\Users\hirok\Documents\Windsurf\811【開発】\youtube_summary_api"
REM Pythonの仮想環境(venv)のactivate.batがあるパスを指定
SET VENV_PATH=%PROJECT_DIR%\.venv\Scripts\activate.bat

REM --- 処理 --- 
echo プロジェクトディレクトリ: %PROJECT_DIR%
echo 仮想環境パス: %VENV_PATH%

REM 指定したディレクトリに移動
cd /d %PROJECT_DIR%

REM 仮想環境を有効化
echo Python仮想環境を有効化しています...
call %VENV_PATH%

REM Tailscale Funnelをバックグラウンドで起動
echo Tailscale Funnelを起動しています...
start "Tailscale Funnel" tailscale funnel 10000

REM FastAPIサーバーを起動
echo FastAPIサーバーを起動しています...
python -m uvicorn main:app --host 0.0.0.0 --port 10000

REM 処理完了後、ウィンドウを閉じる
exit
```

## 自動起動設定手順

        C2("『最上位の特権で実行する』にチェック")

    end

    subgraph D[トリガー]
        D1("『新規』ボタンをクリック")
        D2("タスクの開始を『スタートアップ時』に設定")

    subgraph E[操作]
        E1("『新規』ボタンをクリック")

### ステップ1：タスクスケジューラを開く

1.  `Windowsキー + R` を押し、「ファイル名を指定して実行」ウィンドウを開きます。
2.  `taskschd.msc` と入力し、`Enter`キーを押します。

### ステップ2：タスクの作成

1.  右側の「操作」ペインから `タスクの作成...` をクリックします。

### ステップ3：「全般」タブの設定

1.  **名前**: `YouTube Summary API Auto Start` など、分かりやすい名前を入力します。
2.  **説明**: `PC起動時にFastAPIとTailscale Funnelを起動する` など、タスクの内容が分かる説明を入力します。
3.  **セキュリティオプション**:
    -   `◎ ユーザーがログオンしているかどうかにかかわらず実行する` を選択します。（これにより、PC起動直後、ログイン前からタスクが実行されます）
    -   `☑ 最上位の特権で実行する` にチェックを入れます。

### ステップ4：「トリガー」タブの設定

1.  `新規...` ボタンをクリックします。
2.  「タスクの開始」のドロップダウンメニューから `スタートアップ時` を選択します。
3.  `OK` をクリックします。

### ステップ5：「操作」タブの設定

1.  `新規...` ボタンをクリックします。
2.  「操作」が `プログラムの開始` になっていることを確認します。
3.  「プログラム/スクリプト」の欄に、`start_api.bat` のフルパスを入力します。
    -   例: `C:\Users\hirok\Documents\Windsurf\811【開発】\youtube_summary_api\start_api.bat`
    -   `参照...` ボタンからファイルを選択することも可能です。
4.  `OK` をクリックします。

### ステップ6：「条件」タブの設定

1.  「電源」セクションの `☑ タスクを実行するためにスリープを解除する` のチェックを外します。
2.  「ネットワーク」セクションの `☑ 次のネットワーク接続が使用可能な場合のみタスクを開始する` のチェックを外します。（これにより、オフライン状態でもバッチファイルの実行が試みられます）

### ステップ7：設定完了と保存

1.  すべての設定が終わったら `OK` をクリックしてタスクを保存します。
2.  この時、パスワードの入力を求められる場合があります。お使いの[[Windows]]アカウントのログインパスワードを入力してください。

## 確認方法

設定が完了したら、PCを再起動してください。
ログイン後、しばらくしてからWebブラウザで `http://localhost:10000/docs` にアクセスし、APIのドキュメントページが表示されれば、FastAPIサーバーは正常に起動しています。

また、[[Tailscale]]の管理画面やコマンドラインで、Funnelが有効になっていることを確認してください。

## トラブルシューティング

- **自動起動しない場合**: 
    - タスクスケジューラの対象タスクの「履歴」タブを確認し、エラーが出ていないか確認します。
    - `start_api.bat` を手動でダブルクリックして、正常にサーバーが起動するか確認します。
    - バッチファイル内のパスが正しいか再確認します。
- **コマンドプロンプトのウィンドウが一瞬で消える場合**:
    - `start_api.bat` の最後に `pause` と一行追加して、手動実行した際にエラーメッセージが表示されていないか確認してください。確認後、`pause` は削除してください。

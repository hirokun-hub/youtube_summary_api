# YouTube動画情報取得API プロジェクト概要

---

## 1. このプロジェクトが目指すもの

**目的：**
iPhoneの「ショートカット」アプリからYouTubeのURLを送ると、動画のタイトルや文字起こしなどの情報を自動で取得し、iPhoneに返すバックエンドAPIを構築する。

**解決する課題：**
手作業で行っていたYouTube動画からの情報収集を自動化し、いつでも手軽に動画の重要情報（特に文字起こし）を取得できるようにする。

---

## 2. APIの主な機能

**入力（iPhoneから送るもの）:**
- YouTube動画のURL

**処理（サーバーがやること）:**
1.  リクエストのAPIキーを検証し、正しいリクエストか確認する。
2.  YouTubeから動画の情報を取得する。
    - タイトル、チャンネル名、再生数など
    - **最重要：動画の文字起こしデータ**
3.  取得した情報をJSON形式にまとめて整理する。

**出力（iPhoneに返すもの）:**
- 整理された動画情報のJSONデータ

---

## 3. APIの仕様詳細：データ形式

**リクエスト（入力）の形:**
```json
{
  "url": "https://www.youtube.com/watch?v=xxxxxxxx"
}
```

**レスポンス（出力）の形:**
```json
{
  "title": "動画のタイトル",
  "channel_name": "チャンネル名",
  "video_url": "動画のURL",
  "upload_date": "アップロード日",
  "view_count": "再生数",
  "like_count": "いいね数",
  "subscriber_count": "チャンネル登録者数",
  "transcript": "動画の文字起こし全文..."
}
```
※一部のデータは設定により取得できない場合がある。

---

## 4. セキュリティ：どうやって安全を守るか？

**認証方式：APIキー**

- 許可されたリクエスト（iPhoneショートカット）のみがAPIを利用できるように、「合言葉」となる **APIキー** を使用する。
- リクエスト時にこのAPIキーが送られてこない、または間違っている場合は、処理を拒否してエラーを返す。
- これは、APIのセキュリティ対策として非常に一般的で標準的な手法。

---

## 5. 今回のプロジェクトで「やらないこと」

開発の範囲を明確にし、迅速な実現を目指すため、以下の機能は**含めません**。

-   **AIによる要約機能**:
    文字起こしを要約する機能は作らず、全文をそのまま返す。

-   **ユーザー登録・ログイン機能**:
    APIキーを知っていれば誰でも使えるシンプルな作りにする。

-   **データの保存機能**:
    取得した情報はサーバーに保存せず、iPhoneに返したら処理を完了する。

---

## 6. 使用する技術と公開場所

**プログラミング言語・フレームワーク:**
-   **Python**: 広く使われているプログラミング言語
-   **FastAPI**: Pythonで高速なAPIを簡単に作るためのフレームワーク

**公開（デプロイ）先:**
-   **Render**: 作成したAPIをインターネット上で24時間動かしておくためのクラウドサービス

---

## 7. 課題と今後の展望

**想定される課題：**
-   **長尺動画の処理時間**: 1時間を超えるような長い動画の場合、文字起こしの取得に時間がかかり、iPhoneショートカットがタイムアウト（約30秒）する可能性がある。

**対策の方針：**
1.  まずはシンプルな構成で実装し、実際に問題が発生するか確認する。
2.  問題が顕在化した場合、処理の完了を待たずに一度応答を返し、後から結果を取りに来てもらう「非同期処理」の導入を検討する。（実現可能）

# 02. アプリケーション基盤の構築手順

プロジェクトのセットアップ、お疲れ様でした。GitとGitHubの連携も完了しましたね。
このフェーズでは、APIサーバーの土台となる部分を構築していきます。

## フェーズ2：FastAPIアプリケーションの骨格作成

ここでは、PythonのWebフレームワークである [[FastAPI]] を使って、APIの基本的な形を作ります。

### Step 1: 仮想環境の作成と有効化

プロジェクトごとにPythonの環境を独立させるため、「仮想環境」を作成します。これにより、他のプロジェクトで使っているライブラリとの競合を防ぎます。

ターミナルで `youtube_summary_api` ディレクトリに移動し、以下のコマンドを実行してください。

```bash
# youtube_summary_api ディレクトリに移動
cd C:\Users\hirok\Documents\Windsurf\811【開発】\youtube-summary-tool\まずバックエンドだけ作る\youtube_summary_api

# venvという名前で仮想環境を作成
python -m venv venv

# 仮想環境を有効化
.\venv\Scripts\Activate.ps1
```

> [!NOTE] メモ
> ターミナルのプロンプトの先頭に `(venv)` と表示されれば、仮想環境が有効になっています。

### Step 2: 必要なライブラリのインストール

次に、APIサーバーを動かすために必要なライブラリをインストールします。今回は、APIのフレームワークである `FastAPI` と、サーバーを動かすための `uvicorn` をインストールします。

```bash
pip install fastapi uvicorn
```

### Step 3: "Hello World" APIの作成

動作確認のため、最もシンプルなAPIを作成します。`main.py` という名前のファイルを `youtube_summary_api` ディレクトリ内に作成し、以下の内容を記述してください。

```python:main.py
from fastapi import FastAPI

# FastAPIのインスタンスを作成
app = FastAPI()

# ルートURL ("/") へのGETリクエストを処理する関数
@app.get("/")
def read_root():
    return {"Hello": "World"}
```

### Step 4: APIサーバーの起動

作成したAPIを動かしてみましょう。以下のコマンドを実行すると、開発用のWebサーバーが起動します。

```bash
uvicorn main:app --reload
```

> [!SUCCESS] 確認
> `http://127.0.0.1:8000` にブラウザでアクセスし、`{"Hello":"World"}` と表示されれば成功です！
> サーバーを停止するには、ターミナルで `Ctrl + C` を押してください。

### Step 5: 依存関係ファイルの作成

このプロジェクトで使っているライブラリの一覧を `requirements.txt` というファイルに記録します。これにより、他の環境でも同じライブラリ構成を簡単に再現できます。

```bash
pip freeze > requirements.txt
```

### Step 6: Gitへのコミットとプッシュ

最後に、ここまで作成したアプリケーションの骨格をGitHubに保存します。

```bash
# すべての変更をステージング
git add .

# 変更内容をコミット
git commit -m "feat: Add initial FastAPI application structure"

# GitHubにプッシュ
git push origin main
```

# 01. プロジェクトセットアップ手順

仕様の確認、お疲れ様でした。いよいよ開発の第一歩、プロジェクトのセットアップを始めます。
この資料では、開発を始めるための環境準備の手順を一つずつ解説します。

## フェーズ1：リポジトリの作成とローカル設定

### Step 1: プロジェクト用フォルダの作成

まず、PC上に開発用のフォルダを作成します。
ターミナルを開き、以下のコマンドを実行してください。今回は `youtube_summary_api` というフォルダ名で作成します。

```bash
mkdir ~/youtube_summary_api
cd ~/youtube_summary_api
```

これにより、ホームディレクトリ（`~`）に `youtube_summary_api` という名前のフォルダが作成され、そのフォルダ内に移動します。今後の作業はこのフォルダ内で行います。

### Step 2: 新しいGitHubリポジトリの作成

次に、ソースコードを管理するための新しいリポジトリをGitHub上に作成します。

1.  **[こちらのリンク](https://github.com/new)から、新しいリポジトリ作成ページにアクセスしてください。**
2.  以下の通りに設定します。
    *   **Repository name**: `youtube-summary-api` （先ほどのフォルダ名と合わせると分かりやすいです）
    *   **Description**: （任意）`YouTube動画の情報を取得するAPI` など、分かりやすい説明を記入します。
    *   **Public** を選択します。（誰でもコードを閲覧できますが、書き込みはできません）
    *   **Initialize this repository with:** のセクションでは、**何もチェックを入れないでください**。
        *   `Add a README file`
        *   `Add .gitignore`
        *   `Choose a license`
        *   ↑これら3つは全てチェックを外したままにしてください。後ほど我々で作成します。
3.  **`Create repository`** ボタンをクリックします。

### Step 3: 作成したリポジトリのURLを教えてください

リポジトリが作成されると、"Quick setup" という画面が表示されます。
その画面に表示されている **HTTPS** のURLと **SSH** のURLの両方をコピーして、私に教えてください。

例：
-   HTTPS: `https://github.com/your-username/youtube-summary-api.git`
-   SSH: `git@github.com:your-username/youtube-summary-api.git`

このURLを使って、先ほどローカルに作成したフォルダとGitHubを連携させます。
